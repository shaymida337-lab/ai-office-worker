## 2. הבאגים

### א. קריסת ה-null — **מוגנת במסך הזה** ✅
`formatInvoiceAmount` (1216) מוגן: `amountLabel → "סכום חסר" → formatCurrency→formatAmount`. ה-Metrics (635) וה-month-totals (1202) מסננים `amount>0` לפני. **הקריסה מהיום הייתה ב-Nav.tsx (תוקנה ב-formatAmount).** נשאר סיכון אחד: `invoice.date.slice(0,10)` (215, 242) — הטיפוס אומר `date: string` אבל רשומות GSI/FDR עלולות להגיע בלי תאריך → קריסה. **מוגן ב-`formatInvoiceDate` (1222) אבל לא בסינון.**

### ב. "לא זוהה" והסכומים החסרים — הבעיה האמיתית של המשתמש 🟠
- `MISSING_VALUE="לא זוהה"` משמש ל-**שלושה דברים שונים**: ספק חסר, מספר חשבונית חסר, ותאריך חסר — אותה מילה בדיוק. מבלבל.
- ספקי-זבל (30%) **מסוננים למגירת זבל** (`isJunkInvoice` → junk drawer), אבל ספק ריק/null שאינו מסווג זבל מופיע ברשימה הראשית כ"לא זוהה" סתמי.
- **המלצה:** תוויות מובחנות — `"ספק לא זוהה"` / `"ללא מספר"` / `"ללא תאריך"` (במקום אותו "לא זוהה"), וסכום חסר כבר מציג "סכום חסר". זה שינוי תצוגה בטוח שמפחית בלבול משמעותית.

### ג. באגים/קריסות נוספים
- **מחיקת bulk סדרתית בלי rollback** (482) — כשל באמצע משאיר חלק מחוק. סיכון בינוני.
- **חילוץ id ברגקס** (392, 395, 1252, 1255) — שביר אם פורמט ה-id ישתנה.
- **סינון תאריך client-side בלבד** — `fromDate/toDate` לא נשלחים ל-API (buildListQueryString), אז סינון על נתונים חלקיים שנטענו.
- **`systemNoteForInvoice`** (1275) — 10+ התאמות מחרוזת על `decisionReason`; שביר לשינויי backend (נכשל בשקט).

## 3. עיצוב מול התקן — הסטייה הגדולה במערכת 🔴
- **243 צבעי hex קשיחים** — הדומיננטי `#111827` (85×, אינו הטוקן `#0A0D12`), `#E5E7EB` (33×, גבול לא-טוקן), `#1D4ED8` (13×, כחול שאינו ה-accent `#1D5BFF`).
- **reset CSS inline של ~90 שורות** (513-604): `color:#111827 !important` + `background:#f8fafc !important` על **כל** אלמנט במסך (`.invoice-page-safe :where(...)`). זה ה**שורש** לניתוק מהטוקנים — sledgehammer שנוסף כנראה לתיקון באג ניגודיות/dark-mode, וכופה אפור שאינו בפלטה.
- **סיכון גבוה לתיקון:** ה-`!important` reset הוסף בכוונה; הסרה עיוורת עלולה להחזיר את באג הניגודיות. חייב staging זהיר עם visual-QA.

## 4. סקירה — כפתורים/טפסים/loading/מובייל/RTL
- כפתורים (ייבוא/סריקה/אישור/מחיקה) — עובדים; סריקה מוגנת ב-`disabled={scanning}`.
- מצבי loading: `monthsLoading`, `scanning`, `bulkDeleting`, `loadingMonth` — קיימים.
- מובייל/RTL: יש `InvoiceMobileList` נפרד + `dir="rtl"` — נראה תקין, לוודא ב-visual-QA.
- אישור/מחיקה עם `window.confirm` חוסם — עובד אך לא מעוצב (נמוך).

## תוכנית מדורגת — מה קודם, מה לדחות

1. **הגנת date-null** בסינון (215, 242) + ב-KPI (`new Date(invoice.date)`): fallback בטוח. **סיכון: נמוך מאוד.**
